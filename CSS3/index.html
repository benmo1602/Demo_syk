CSS3简介

CSS3参考手册[推荐：https://www.css88.com/book/css/quicksearch.htm]
CSS3是CSS的升级版本，3是版本号，目前主流浏览器都支持CSS3，除了IE9及以下版本
为了知道浏览器是否支持某个css属性，可以通过 Can I Use ... ?（http://caniuse.com）这个网站进行查询
不同浏览器需要不同的前缀，它表示该CSS属性或规则并未成为W3C标准，是浏览器的私有属性，为了良好的兼容性，最好加上前缀：
  • -webkit-：Chrome和Safari浏览器
  • -moz-：firefox浏览器
  • -ms-：IE浏览器
  • -o-：Opera浏览器
（1）浏览器兼容性机制：比如想写：background: linear-gradient(90deg, yellow, red);那么，兼容性写法如下，利用css的层叠生效机制把标准格式放在最下方，这样能确保标准的覆盖浏览器自带的
  • background: -webkit-linear-gradient(0deg, yellow, red);
  • background: -moz-linear-gradient(0deg, yellow, red);
  • background: -ms-linear-gradient(0deg, yellow, red);
  • background: -o-linear-gradient(0deg, yellow, red);
  • background: linear-gradient(90deg, yellow, red);
（2）浏览器回退机制：也是利用CSS的层叠生效机制，即在下方的会覆盖上方的，如果下方的不生效，就使用上方能生效的。为的是保证在低版本浏览器中网站也能运行，不至于完全挂掉。比如设置渐变时，在上方放一个非渐变
  • background: rgb(255, 128, 0);
  • background: -moz-linear-gradient(0deg, yellow, red);
  • background: -o-linear-gradient(0deg, yellow, red);
  • background: -webkit-linear-gradient(0deg, yellow, red);
  • background: linear-gradient(90deg, yellow, red);
另外也可以利用辅助类，或者css新特性@support，但慎用，因为好多不支持

  • @supports (text-shadow: 0 0 .3em gray) {

CSS3动画

（1）过渡（transition，这是个复合属性，即把好几个属性写在这一个里，利用这个可以实现响应式动画）
  • transition-property：指明哪些CSS属性变化时需要过渡
  • transition-duration：完成过渡需要的时间
  • transition-timing-function：过渡函数，默认只有5种，常用ease-in-out
  • transition-delay：延迟多久才执行过渡
（2）关键帧（keyframes，用于定义动画，指定动画每个步骤时的样式，常以@keyframes 动画名{}）
（3）动画（animation，这是个复合属性，即把好几个属性写在这一个里，利用这个可以实现动画）
  • animation-name：通过keyframes关键帧定义的动画的名称
  • animation-duration：完成一次动画需要的时间
  • animation-timing-function：动画的播放方式，默认只有5种，常用ease-in-out
  • animation-delay：延迟多久才执行动画
  • animation-iteration-count：动画的执行次数，无限次为infinite
  • animation-direction：每次动画的播放顺序，normal代表每次都是从0%到100%，alternate代表偶数次时0%到100%，奇数次时100%到0%
  • animation-play-state：控制动画的播放状态，running播放，paused暂停
  • animation-fill-mode：动画开始前和动画结束后要执行的操作（none:结束后回到开始时的样式，forwards：结束后保持结束时的样式，backwards:动画每次播放时都从初始帧开始，both：同时具有forwards和backwards的效果）

CSS3媒体查询（media queries）

（1）背景：随着科学技术的发展，网页的浏览终端越来越多样化，目前有以下5种基本的浏览终端：
  • 宽屏电视
  • 台式电脑
  • 笔记本电脑
  • 平板电脑
  • 智能手机
尽管无法保证在每一种浏览终端所看到的网页都一模一样，但至少要使网页能适配用户的浏览终端，而使用CSS3的media queries媒体查询功能就可以实现一个网页适配不同的终端
（2）媒体类型（media type，在CSS2中很常见也很有用的属性，可以通过媒体类型对不同的设备指定不同的样式），目前有以下10种：
  • all：所有设备
  • screen：电脑显示器
  • print：打印用纸或打印预览视图
  • braille：盲人用点字触觉反馈设备
  • embossed：盲文打印机
  • handheld：便携设备
  • projection：各种投影设备
  • speech：语音或音频合成器
  • tv：电视机类型设备
  • tty：使用固定密度字母栅格的媒介，比如电传打字机和终端
（3）媒体类型的使用方法：
  • link标签的media属性： href="./style.css" media="screen"
  • @importurl方法： @importurl(./style.css) screen;
  • CSS3新引进的@media方法： @media screen{我们的样式}
（4）媒体查询的使用方法：
  • 基本结构： @media 媒体类型 and ( 媒体特性 ) { 我们的样式 }
  • 多个媒体特性： @media only print and (max-width: 1200px) and (min-width: 800px){样式代码}
  • only关键字： @media only print and (max-width: 1200px){样式代码}
  • not关键字： @media not print and (max-width: 1200px){样式代码}

CSS3响应式设计

（1）效果：响应式设计能让网页在不同的设备中展现不同的设计风格，前提是：
  • 网页有灵活的网格基础： 流体网格：即每个网格格子都是以百分比控制的大小，这样就能让网格随时根据屏幕按比例缩放
  • 引用的图片必须是可伸缩的： 弹性图片：图片大小不是固定的，也是百分比，如img{max-width:100%}
  • 不同的显示风格，需要在媒体查询media queries中写入不同的样式： 主要断点：如@media screen and (max-width:1000px){}
（2）技巧：
  • 尽量少用无关紧要的div
  • 少用内联元素inline
  • 少用js或flash
  • 少用绝对定位和浮动样式
  • 摒弃：任何冗余结构和不使用100%设置
  • 使用HTML5 Doctype和相关指南
  • 重置好样式，即把浏览器默认的样式设为我们想要的样式，避免浏览器的影响
  • 一个简单的有语义的核心布局
  • 给重要的网页元素使用简单的技巧，比如导航菜单之类元素
  • 当图片（或其他元素）以行列式进行布局时，让视口的宽度来决定列的数量。弹性盒布局（即 Flexbox）或者 display: inline-block加上常规的文本折行行为，都可以实现这一点。
  • 在使用多列文本时，指定 column-width （列宽）而不是指定column-count （列数），这样它就可以在较小的屏幕上自动显示为单列布局
（3）meta标签：在响应式设计中，可视区域meta标签是不可或缺的，meta name=”viewport” content=””，content的内容如下：
  • width：可视区域的宽度或者device-width
  • height：可视区域的高度或者device-height
  • initial-scale：初始缩放比例，默认为1，即没有缩放
  • maximun-scale：最大缩放比例
  • minimun-scale：最小缩放比例
  • user-scalable：用户是否可以缩放可视区域，yes、no

（10）伪随机背景：如果定义了多组背景图，且背景图之间有重叠，写在前面的将会盖在写在后面的图像之上。利用设置多背景时，最开始的背景（在最顶层）会遮住后面的背景的原理，实现遮挡效果，并使用【最小公倍数】最大化规则，即使用【相对质数】，使背景的重复性降低，从而达到模拟自然界的随机背景。
